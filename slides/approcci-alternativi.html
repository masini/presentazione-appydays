JHipster

JHipster is a Yeoman generator, used to create a Maven + Spring + AngularJS project, with full hot reload of Java and JavaScript code.

JHipster ha tutto ciò che ci serve

  • Un generatore Yeoman in grado di metterci a disposizione un progetto Maven/Gradle con:
    • Ecosistema di build Web (npm/bower/grunt) integrato con Maven/Gradle
    • Runtime Grunt integrato con Runtime Java

JHipster ha tutto ciò che ci serve

  • Un agent (evoluzione di Spring Loaded) in grado di:
    • Ricaricare classi/metodi/field
    • Aggiornare i Service/Component di Spring
    • Modificare l'EntityManager alla modifica di un entity
    • Mappare le nuove URL Spring MVC REST
    • Flushare la cache di Jackson se modifichiamo un POJO

Ma allora perchè non usarlo ?

USATELO !!

  • Se il vostro framework di riferimento è Spring
  • Se state facendo un prodotto e non un applicativo interno
  • Se avere un uso di circa 100MB di Permanent Generation non è un problema (beati voi !!)

Quando non ne vale la pena ?

  • Se il vostro framework di riferimento è Java EE >=6
  • Se le vostre operation non sono abbastanza "DevOps"
  • Se sul vostro Application Server girano abbastanza applicazioni da non potervi permettere tutte le librerie che si porta dietro Spring

 

O piú semplicemente perchè non sopportate i baffetti da Hipster !!!
