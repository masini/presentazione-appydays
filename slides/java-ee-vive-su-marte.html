Java EE viene da Marte...

Nonostante l'opera di svecchiamento iniziata con Java EE 5, Java sul server sente gli anni del linguaggio e della piattaforma

Chi vive il mondo Java Enterprise,

Il linguaggio Java...

  • È ormai consolidato da anni di sviluppo e dalla presenza più pervasiva presso le grandi aziende dopo il COBOL
  • Questo ne ha fatto la sua fortuna, ma lo ha reso molto "resistente" al cambiamento
  • Tanto per avere un'idea....

C#

Forse l'unico vero competitor enterprise di Java

  • Si è evoluto tantissimo nei suoi poco piú di 10 anni di vita
    • È stato reso funzionale
    • Partial classes per supportare i generatori di codice
    • Integrato con un sistema di interrogazione dei dati
    • L'ultima versione ha anche supporto per async/await

La piattaforma Enterprise...

  • Gira su application server "monumentali" che ricordano le architetture mainframe
  • È vincolata al deploy durante lo sviluppo, e quindi ad attese di secondi/minuti ad ogni modifica
  • Tra i vari standard JSR che la compongono, non ne ha nessuno realmente competivo lato front-end
  • Come "innovazione" in Java EE 8 avremo Spring MVC !!! (JSR 371)

Java Server Faces

È lo standard di diritto per lo sviluppo Web in Java EE

  • Abbiamo l'impressione di scrivere HTML5 con l'XML delle Faces...
  • ...è ricco di componenti già pronti (specialmente con PrimeFaces)
  • Questo lo rende molto adatto allo sviluppo Enterprise....
  • ...finchè non deve essere fatta qualche personalizzazione...

Google Web Toolkit

Ci permette di usare lo stesso linguaggio sia lato client che lato server !!

  • Il linguaggio è lo stesso ma le API no
  • I tempi di compilazione vanno oltre la salute mentale dei comuni mortali
  • Anche se compila JavaScript è un mondo chiuso, una scelta di campo che impedisce una facile integrazione con l'ecosistema Web.
